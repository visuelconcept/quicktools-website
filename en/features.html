Features

Three complementary components for communication, synchronization and data persistence.

QuickSERVER — Message Broker

Publish/subscribe messaging system for real-time communication between distributed applications.

Datagroups

Message routing by named groups. Clients subscribe to relevant groups for loose coupling.

Typed messages

Automatic binary serialization of native types: char, short, long, float, double, string. Cross-architecture compatible.

Compression

Transparent message compression to optimize network bandwidth.

Load balancing

Multiple load distribution strategies: round-robin, weighted, best-effort, single recipient or broadcast.

Timer & periodic messages

QuickTIMER enables scheduled event propagation at regular intervals.

HTTP Dashboard

Built-in web interface (port 1251) for real-time server monitoring.

Message acknowledgment

Delivery confirmation with configurable timeouts and guaranteed delivery.

Automatic reconnection

Reconnection modes: manual, automatic or context-aware, for maximum resilience.

Server interconnection

Router mode for distributed multi-site architectures with inter-server message propagation.

QuickSDB — Shared Database

Real-time shared variable system, automatically synchronized across all connected clients.

Typed variables

Boolean, byte, short, long, float, double, date, time, delay, string, enum, object and collection.

Transparent synchronization

Every modification is instantly propagated to all subscribed clients.

Object model

Class definitions with inheritance and typed members for structured data organization.

Algebraic formulas

Variables computed automatically through if-then-else expressions.

Persistence

Save and load the complete database state to/from disk.

Transactions

Commit/rollback support for atomic and consistent modifications.

Watch / Unwatch

Selective variable monitoring by patterns for efficient event-driven processing.

Variable groups

Bulk operations and monitoring of related variable groups.

DbEngine — SQL Abstraction

Unified API for 6 relational database engines. Switch engines without modifying your code.

Engine Type Typical use
SQLite Embedded Default, no installation required
MySQL Server Web and general-purpose applications
PostgreSQL Server Complex queries, SQL compliance
Oracle Enterprise Large systems, high availability
MS SQL Server Enterprise Microsoft ecosystem
Sybase Enterprise Legacy systems, finance

Code example

Sending and receiving a message with QuickSERVER in C:

/* Send a message */
T_DATAGROUP dg = QT_SocketJoinDG(sock, "PRODUCTION");

T_MSG msg;
QT_SocketInitMsg(&msg);
QT_SocketPutLong(&msg, "sensor_id", 42);
QT_SocketPutDouble(&msg, "temperature", 23.5);
QT_SocketPutString(&msg, "status", "OK");

QT_SocketSendMsg(sock, dg, &msg);

/* Receive */
if (QT_SocketReadMsg(sock, &msg, 5000) == QT_OK) {
    long id   = QT_SocketGetLong(&msg, "sensor_id");
    double temp = QT_SocketGetDouble(&msg, "temperature");
}
