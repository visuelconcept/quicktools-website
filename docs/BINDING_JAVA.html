Binding Java — QuickSERVER et QuickSDB

Ce document decrit les bindings Java fournis avec QuickTOOLS 2.81 : le package visuelconcept.quickserver pour la communication et le package visuelconcept.quicksdb pour la base de donnees partagee.

QuickSERVER (package visuelconcept.quickserver)

Le binding Java utilise JNI pour acceder a la bibliotheque native QuickSERVER.

Classes principales

Classe Description
QuickSERVER Classe statique — connexion, reception, monitoring
DataGroup Canal de communication (envoi/reception)
Message Message ASCII ou binaire, avec serialisation typee

Classes internes (constantes)

Classe Description
QuickSERVER.Delivery Modes de delivrance : BEST, ONE, ALL, ONEWAIT, ALLWAIT
QuickSERVER.Balancing Modes d'equilibrage : NONE, ROUND_ROBIN, WEIGHTED, SORTED
QuickSERVER.Errno Codes d'erreur (0-24)
QuickSERVER.ShutdownMode IMMEDIATELY, AFTERREADINGALL_MESSAGES
QuickSERVER.ReconnectionMode NONE, AUTO, CONTEXT

Fichiers

Fichier Role
SERVER/java/sources/visuelconcept/quickserver/QuickSERVER.java Classe principale

SERVER/java/sources/visuelconcept/quickserver/DataGroup.java DataGroups
SERVER/src/client-java.cpp Implementation JNI

Bibliotheque native

Le binding Java charge la bibliotheque native via System.loadLibrary() :

lib_QTsocket  (essai principal)
_QTsocket     (fallback)

IMPORTANT : le nom de cette bibliotheque ne doit jamais etre modifie sous peine de casser tous les clients Java existants.

Installation et classpath

Apres l'installation de QuickSERVER (via l'installeur ou le build), les fichiers Java se trouvent dans le repertoire %QT_HOME%\lib\Java\ :

Fichier Description
quickserver.jar Classes Java QuickSERVER (Message, DataGroup, QuickSERVER)
quicksdb.jar Classes Java QuickSDB
lib_QTsocket.dll Bibliotheque native JNI pour QuickSERVER
lib_QTsdb.dll Bibliotheque native JNI pour QuickSDB

Compilation d'un programme Java

javac -classpath "%QT_HOME%\lib\Java\quickserver.jar;%QT_HOME%\lib\Java\quicksdb.jar" MonClient.java

Execution

java -classpath "%QT_HOME%\lib\Java\quickserver.jar;%QT_HOME%\lib\Java\quicksdb.jar;." -Djava.library.path="%QT_HOME%\lib\Java" MonClient

Le parametre -Djava.library.path est necessaire pour que Java trouve les DLLs JNI natives.

Construction des JARs depuis les sources (developpement)

cd port2.6\SERVER\java
build_jar.bat

cd port2.6\SDB\java
build_jar.bat

Les JARs sont generes dans port2.6\build\lib\Java\.

Utilisation rapide

import visuelconcept.quickserver.*;
import java.io.IOException;

public class MonClient {
    public static void main(String[] args) {
        try {
            // Connexion
            QuickSERVER.init(null, "MonClient", QuickSERVER.ReconnectionMode.AUTO);

            // Envoi d'un message ASCII
            DataGroup dg = QuickSERVER.getDG("mon_sujet");
            dg.sendMsg(1001, "Bonjour");

            // Reception
            DataGroup dg_recv = QuickSERVER.joinDG("mon_sujet");
            Message m = QuickSERVER.readMsg();

            if (m.isBinary()) {
                int val = m.readInt();
                double x = m.readDouble();
                String s = m.readString();
            } else {
                String texte = m.getAsciiData();
            }

            // Reponse directe a l'emetteur
            m.getSource().sendMsg(1002, "Bien recu");

        } catch (IOException ex) {
            System.err.println("Erreur: " + ex.getMessage());
        }

        QuickSERVER.terminate();
    }
}

Utilisation rapide

import visuelconcept.quicksdb.*;
import java.io.IOException;

public class MonClientSdb {
    public static void main(String[] args) {
        try {
            // Connexion a QuickSDB (instance 0)
            QuickSDB.connect(null, 0);

            // Ajouter des variables
            QuickSDB.addVariable(0, "temperature", QuickSDB.FieldType.DOUBLE, 20.0, null);
            QuickSDB.addVariable(0, "alarme", QuickSDB.FieldType.BOOLEAN, 0, null);

            // Formule : alarme = temperature > 30
            QuickSDB.addFormula(0, null, "alarme", "temperature > 30", null, null);
            QuickSDB.commit(0);

            // Ecrire et evaluer
            QuickSDB.writeNumericVariable(0, "temperature", 35.5);
            QuickSDB.commit(0);
            QuickSDB.evaluateAllFormulas(0);
            QuickSDB.commit(0);

            // Lire les resultats
            double temp = QuickSDB.readNumericVariable(0, "temperature");
            double alarm = QuickSDB.readNumericVariable(0, "alarme");

            System.out.println("temperature=" + temp + " alarme=" + alarm);

            // Persistence
            QuickSDB.save(0, "ma_base.ddl");
            QuickSDB.disconnect(0);

        } catch (IOException ex) {
            System.err.println("Erreur SDB: " + ex.getMessage());
        }
    }
}

Notes de migration 2.81

1. Activation des methodes setMsg* (priorite, delivrance, timeout, equilibrage)

Les 4 methodes suivantes etaient declarees mais commentees dans le code Java, et leur implementation JNI etait un stub (no-op). Elles sont maintenant fonctionnelles :

  • QuickSERVER.setMsgPriority(short priority) — Definit la priorite par defaut
  • QuickSERVER.setMsgDelivery(int delivery) — Definit le mode de delivrance
  • QuickSERVER.setMsgTimeout(short timeout) — Definit le timeout d'acquittement
  • QuickSERVER.setMsgBalancing(int balancing) — Definit le mode d'equilibrage

Pourquoi ces methodes ont ete ajoutees :

L'API C permet de modifier directement les champs de TMESSAGEHEADER sur chaque message avant l'envoi (priorite, delivrance, timeout, equilibrage). Le binding Java n'expose pas directement cette structure d'en-tete. Ces 4 methodes fournissent un controle equivalent en definissant des valeurs par defaut appliquees automatiquement a chaque message envoye.

Chaque methode retourne la valeur precedente, permettant de la restaurer :

// Passer en delivrance garantie pour un envoi critique
int old = QuickSERVER.setMsgDelivery(QuickSERVER.Delivery.ONE_WAIT);
dg.sendMsg(1001, "message important");
QuickSERVER.setMsgDelivery(old);  // restaurer

Impact sur les clients existants : Aucun. Ces methodes n'etaient pas accessibles auparavant (commentees).

2. Modernisation de la compilation Java

  • javah -jni remplace par javac -h (compatible JDK 10+)
  • -target 1.1 remplace par -source 8 -target 8
  • Makefiles remplaces par des scripts .bat

Impact : les clients doivent utiliser JDK 8 minimum pour recompiler. Les JAR existants compiles avec un ancien JDK continuent de fonctionner.

3. Nouveau binding Java pour QuickSDB

Le package visuelconcept.quicksdb est nouveau dans la version 2.81. La classe QuickSDB etait auparavant un placeholder vide. Elle est maintenant completement implementee avec 50+ methodes natives couvrant l'integralite de l'API C SDB.

Impact : Aucun. La classe etait vide et inutilisable.

Limitations connues

Limitation Detail
JNI pointers 32-bit Les pointeurs C sont stockes dans des int Java (32 bits). Fonctionne en x86, incompatible x64 natif. Pour x64, il faudrait migrer vers long Java.
finalize() Java Les classes Message et DataGroup (QuickSERVER) utilisent finalize() pour le nettoyage. Cette methode est deprecie depuis Java 9. Un mecanisme Cleaner devrait etre envisage pour les futures versions.
