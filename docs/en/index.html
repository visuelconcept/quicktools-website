QuickTOOLS 2.81 — Technical Documentation

Overview

QuickTOOLS is a middleware framework developed by Visuel Concept SAS since 1994. It provides a real-time client-server communication infrastructure, a distributed shared database and multi-database SQL access — all in native C for Windows.

Key Concepts

Datagroups (SERVER)

A datagroup is a named communication channel. Clients subscribe (join) to a datagroup to receive messages published to it. This is a publish/subscribe model:

T_DATAGROUP dg = QT_SocketJoinDG("PRODUCTION");
QT_SocketSendMsg(msg, dg);  /* all subscribers receive the message */

Typed Messages (SERVER)

Each message has a type (integer >= 1000 for user messages) and can contain serialized binary data or ASCII text. The binary format allows writing/reading typed fields (char, short, long, float, double, string) in a portable way across architectures.

Shared Variables (SDB)

QuickSDB manages a space of typed variables synchronized in real time:

QT_SdbAddVariable(instance, "TEMPERATURE", C_DOUBLE, 0.0);
QT_SdbSetDouble(instance, "TEMPERATURE", 42.5);
QT_SdbWatchVariable(instance, "TEMPERATURE", callback, arg);

Changes are automatically propagated to all subscribed clients.

SQL Abstraction (DbEngine)

A single API to access 6 different databases:

T_SQL *db = Sql_Init("sqlite");
Sql_Connect(db, NULL, "mabase.db", NULL, NULL);
T_SQL_RESULT *r = Sql_Execute(db, "SELECT * FROM clients WHERE nom = '%s'",
                               Sql_String(db, nom));
while (Sql_Fetch(r)) {
    T_SQL_FIELD *f = Sql_GetField(r, "nom", 0);
    printf("%s\n", f->Value.String);
}
Sql_FreeResult(r);
Sql_Disconnect(db);
Sql_Close(db);

Deliverable Formats

Architectures

Libraries and executables are available in two architectures:

Architecture Target Compiler flag
x86 32-bit Windows (default)
x64 64-bit Windows /D_WIN64 /DWIN64

Library Types

Each library (qtutil, qtserver, qtsdb, qtdbengine) is available in two formats:

Format Extension Description Usage
Static .lib Code embedded in the executable at link time Standalone applications, no DLL dependency
Shared (DLL) .dll + .lib (import) Code loaded dynamically at runtime Memory sharing between modules, update without recompilation

C Runtime (multithread linking)

The choice of C runtime determines how your application links to the Microsoft standard library:

Option Flag Runtime Linked library
Multithread DLL (default) /MD Shared runtime (msvcrt.dll) msvcrt.lib
Multithread static /MT Runtime embedded in the executable libcmt.lib

Both modes are multithread (/D_MT). The difference is:

  • /MD: smaller executable, requires msvcrt.dll (present on all Windows systems)
  • /MT: standalone executable, no external C runtime dependency

Important: All modules linked in the same executable must use the same runtime (/MD or /MT). Do not mix the two.

License Management

QuickTOOLS 2.81 uses Cryptlex for license management. The command-line tool QuickActivator.exe allows activating, deactivating and checking licenses.

Network Requirements and Communication

The Cryptlex license system establishes outgoing HTTPS connections to Cryptlex servers (api.cryptlex.com) for:

  • License activation / deactivation
  • Periodic validity check (at each server startup and every 30 minutes during execution)
  • Synchronization of expiration dates and subscription renewals
  • Verified trial activation

Ports and protocols:

Protocol Port Direction Usage
HTTPS 443 Outgoing Communication with api.cryptlex.com

If the PC does not have internet access (isolated network, firewall), the system operates in degraded mode:

  • Local trial (90 days) instead of verified trial
  • Offline activation via file exchange (see scenario 5)
  • Grace period for already activated licenses (the license remains valid for a certain time without contacting the Cryptlex server)

> Firewall: If a proxy or firewall filters outgoing connections, allow HTTPS access (port 443) to api.cryptlex.com.

License Storage

License data is stored locally on the machine by the Cryptlex SDK:

Data Location Description
Activation data Windows Registry (HKCU) License key, machine fingerprint, activation tokens
Local trial data Windows Registry (HKCU) Trial start date, remaining duration
Product.dat Embedded in the executable Product public key (cryptographic verification)

License data is bound to the machine (hardware fingerprint). It is not transferable by simply copying files — use the transfer procedure (deactivation + reactivation).

> Note: No license.dat file is needed. The Cryptlex system replaces the old file-based license system used in previous versions of QuickTOOLS.

License Types

Type Description
Trial Time-limited evaluation license, with automatic server shutdown after 2 hours of continuous execution. If no license is found at startup, QuickSERVER automatically starts in trial mode.
Subscription Fixed-duration license, renewable by subscription
Perpetual Permanent license with no expiration date

Features

Licenses are associated with features that determine which products are accessible:

Feature Product
server QuickSERVER — message server
sdb QuickSDB — shared database

Online Activation

QuickActivator.exe activate XXXX-XXXX-XXXX-XXXX

Offline Activation (PC not connected to the internet)

  1. Set the license key (will fail because there is no internet):
    2.    QuickActivator.exe activate XXXX-XXXX-XXXX-XXXX
  2. Generate an activation request:
    3.    QuickActivator.exe offline-request activation_request.dat
  3. Transfer the activation_request.dat file to a connected PC and upload it to the Cryptlex customer portal
  4. Download the response file and transfer it to the target PC
  5. Apply the response:
    6.    QuickActivator.exe offline-activate activation_response.dat

Deactivation

QuickActivator.exe deactivate

For offline deactivation:

QuickActivator.exe offline-deactivate deactivation_request.dat

Check Status

QuickActivator.exe status

Automatic Trial Mode at Startup

If no valid license is found at startup, QuickSERVER automatically attempts:

  1. A verified trial activation (requires internet)
  2. A local trial (without internet, 90 days)

In trial mode, server execution is limited to 2 continuous hours. Restart the server to resume.

Periodic License Verification

QuickSERVER checks license validity every 30 minutes during execution. This check applies to all modes (trial, subscription, perpetual) and enables:

  • Hot activation: activate a paid license with QuickActivator.exe while the server is running in trial mode — the 2-hour limit is automatically removed without restart.
  • Hot renewal: when a subscription is renewed, the server updates the status automatically.
  • Expiration detection: if the license expires or is revoked during execution, the server shuts down gracefully.

Security rules for trial mode:

The following operations require a mandatory server restart to take effect:

Operation Effect without restart After restart
Trial expired (2h reached) Server stopped New 2-hour cycle
trial-extend (extension) No effect on the current session New calendar duration taken into account
License downgraded to trial (subscription expiration) Server stopped Restarts in trial mode (2h)

This prevents indefinitely extending a trial session without restarting the server.

> Summary: only the trial to paid license transition is handled hot (without restart). All other transitions involving a trial require a restart.

Status changes are logged in the server logs.

Usage Scenarios

Scenario 1: First Launch (Evaluation)

You have just installed QuickSERVER and do not yet have a license.

  1. Launch QuickSERVER normally:
    2.    bin\QuickServer.exe
  2. The server detects the absence of a license and automatically activates trial mode.

- If the PC is connected to the internet: verified trial via Cryptlex - If the PC is offline: local trial (90 days maximum)

  1. The server operates normally with all features enabled (server + sdb).
  2. Limitation: the server automatically stops after 2 hours of continuous execution. Restart it to resume.

> Tip: Trial mode allows you to test all features before purchasing a license.

Scenario 2: Upgrading from Trial to a Perpetual License

You are using trial mode and wish to purchase a permanent license.

  1. Obtain a perpetual license key from Visuel Concept.
  2. Activate the license (the trial is automatically replaced):
    3.    bin\QuickActivator.exe activate XXXX-XXXX-XXXX-XXXX
  3. Check the status:
    4.    bin\QuickActivator.exe status

The output shows Type: perpetual and Expiration: perpetuelle.

  1. The server automatically detects the new license within 30 minutes and removes the 2-hour limit. No restart required.

> Note: It is not necessary to deactivate the trial before activating a license. Activation automatically replaces the previous mode.

Scenario 3: Upgrading from Trial to a Subscription License

  1. Obtain a subscription license key from Visuel Concept.
  2. Activate the license:
    3.    bin\QuickActivator.exe activate XXXX-XXXX-XXXX-XXXX
  3. Check the status:
    4.    bin\QuickActivator.exe status

The output shows Type: subscription with the expiration date and the number of remaining days.

  1. The server automatically detects the new license within 30 minutes and removes the 2-hour limit. No restart required.

Scenario 4: Subscription Renewal

Your subscription is nearing expiration.

  1. Renew your subscription with Visuel Concept (the license key remains the same).
  2. The periodic check (every 30 minutes) automatically updates the expiration date from Cryptlex (requires internet). No restart required.
  3. Verify the renewal:
    4.    bin\QuickActivator.exe status

The expiration date and the number of remaining days are updated.

> Offline PC: If the PC does not have internet access, the license remains valid during a grace period. Connect the PC to the internet before the grace period expires to synchronize the renewal.

Scenario 5: Offline Activation (PC without internet)

The target PC is not connected to the internet.

  1. On the target PC — attempt activation (stores the key locally):
    2.    bin\QuickActivator.exe activate XXXX-XXXX-XXXX-XXXX

Online activation fails, but the key is saved.

  1. On the target PC — generate the request file:
    2.    bin\QuickActivator.exe offline-request activation_request.dat
  1. Transfer the activation_request.dat file to a connected PC (USB drive, internal network, etc.).
  1. On the connected PC — go to the Cryptlex customer portal:

- Upload the activationrequest.dat file - Download the response file activationresponse.dat

  1. Transfer the activation_response.dat file to the target PC.
  1. On the target PC — apply the response:
    2.    bin\QuickActivator.exe offline-activate activation_response.dat

If the server is running, the license is detected automatically within 30 minutes. No restart required.

  1. Verify the activation:
    2.    bin\QuickActivator.exe status

Scenario 6: Transferring a License to Another PC

You are changing machines and wish to move your license.

  1. On the old PC — deactivate the license:
    2.    bin\QuickActivator.exe deactivate

This frees the activation slot on Cryptlex. If QuickSERVER is running, it detects the deactivation within 30 minutes and stops automatically.

  1. On the new PC — activate the same license key:
    2.    bin\QuickActivator.exe activate XXXX-XXXX-XXXX-XXXX

> Old PC offline? Use offline deactivation: > ``batch > bin\QuickActivator.exe offline-deactivate deactivationrequest.dat > ` > Then upload deactivationrequest.dat` to the Cryptlex portal from a connected PC.

Scenario 7: License Expired During Execution

Your subscription license expires while the server is running.

  1. The periodic check detects the expiration and gracefully shuts down the server, logging a warning message.
  2. To restore unrestricted operation:

- Renew your subscription with Visuel Concept, then restart QuickSERVER (the license is automatically updated if the PC is connected) - Or purchase a perpetual license and activate it:

     bin\QuickActivator.exe activate NEW-KEY-XXXX-XXXX

Scenario 8: Extending a Local Trial (Offline)

A user on a local trial (90 days) needs more evaluation time.

  1. Extend the trial by an additional 30 days:
    2.    bin\QuickActivator.exe trial-extend 30

The command is cumulative — it can be called multiple times.

  1. Restart QuickSERVER for the extension to take effect. The extension modifies the trial's calendar duration (number of remaining days) but does not reset the current 2-hour session.

> Note: This command only works for local trials (offline). Verified trials (online) are managed by the Cryptlex server.

Scenario 9: Hot Activation of a Paid License (Server in Production)

The server is running in trial mode or with an existing license. You wish to activate a paid license without interrupting service.

  1. Activate the new license while the server is running:
    2.    bin\QuickActivator.exe activate XXXX-XXXX-XXXX-XXXX
  1. Within 30 minutes, the server detects the change and displays in the logs:
    2.    Licence: changement de statut detecte (type: perpetual)
   Licence activee: limite de 2 heures supprimee.
  1. The server continues running without interruption with the new license.

> Important: Hot activation only works for upgrading from a lower mode to a higher mode (trial to paid license, or changing paid license). Downgrading to trial mode (subscription expiration) causes the server to stop and requires a restart.

> Tip: To verify that the change has been taken into account on the server side, check the HTTP dashboard: http://localhost:1251 — the "License Information" section reflects the current status.

Scenario 10: Checking Available Features

Your license may include one or more features (server, sdb).

bin\QuickActivator.exe status

The output indicates the enabled features:

  • Feature server: OUI — QuickSERVER is authorized
  • Feature sdb: OUI — QuickSDB is authorized

In trial mode, all features are enabled by default.
