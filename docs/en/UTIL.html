UTIL — Utility functions and data structures

Overview

The UTIL module is the fundamental foundation of the QuickTOOLS library. It provides a comprehensive set of utility functions covering memory management, string manipulation, date and time operations, configuration, compression, dynamic library loading, internationalization, as well as three generic data structures: the queue (queue), the symbol table (symtable) and the hash table (hashtable).

Headers:

  • QT_std.h — Standard types and basic macros
  • QT_util.h — Main utility functions
  • queue.h — Generic queue
  • symtable.h — Symbol table (balanced binary tree)
  • hashtable.h — Hash table
  • license.h — License management

Library: qt_util.lib

2. Initialization

QT_Init

void QT_Init(int argc, char *argv[]);

Initializes the QuickTOOLS framework. This function must be called before any other QuickTOOLS function. It parses command-line arguments and initializes internal structures (memory, configuration, logging).

Parameters:

  • argc — Number of command-line arguments
  • argv — Array of arguments

QT_Terminate

void QT_Terminate(long status);

Cleanly terminates the QuickTOOLS framework and exits the program. Releases all internal resources, closes open files and calls exit(status).

Parameters:

  • status — Program return code (0 = success)

3. Memory management

Standard functions

QT_Free

void QT_Free(void *Addr);

Frees a memory block previously allocated by QTMalloc or QTRealloc. Wraps free() with optional tracking in debug mode.

Parameters:

  • Addr — Pointer to the memory block to free

QT_Malloc

void *QT_Malloc(size_t Size);

Allocates a memory block of the specified size. Wraps malloc() with error checking and optional tracking in debug mode.

Parameters:

  • Size — Size in bytes of the block to allocate

Return:

  • Pointer to the allocated block, or NULL on failure

QT_Realloc

void *QT_Realloc(void *Addr, size_t Size);

Reallocates an existing memory block with a new size. The content is preserved up to the minimum of the old and new sizes.

Parameters:

  • Addr — Pointer to the existing block
  • Size — New size in bytes

Return:

  • Pointer to the reallocated block, or NULL on failure

Debug memory functions

These functions are only available when the program is compiled with the DEBUG flag. In release mode, they are replaced by the standard functions via macros.

QT_dbgMalloc

void *QT_dbgMalloc(char *File, size_t Size);

Debug version of malloc. Records the source file and call line for allocation tracking.

Parameters:

  • File — Source file and line number (automatically filled by the macro)
  • Size — Size in bytes to allocate

Return:

  • Pointer to the allocated block

QT_dbgFree

void QT_dbgFree(char *File, void *Addr);

Debug version of free. Validates the pointer and updates the allocation tracking.

Parameters:

  • File — Source file and line number
  • Addr — Pointer to the block to free

QT_dbgRealloc

void *QT_dbgRealloc(char *File, void *Addr, size_t Size);

Debug version of realloc. Records the reallocation for tracking.

Parameters:

  • File — Source file and line number
  • Addr — Pointer to the existing block
  • Size — New size in bytes

Return:

  • Pointer to the reallocated block

QT_dbgReportLeaks

void QT_dbgReportLeaks(void);

Displays a report of all memory leaks detected since the last call to QT_dbgMark. Each leak is identified by the source file, line and size of the unreleased block.

QT_dbgMemStats

void QT_dbgMemStats(void);

Displays memory statistics: total number of allocations, deallocations, total allocated size, peak memory usage.

QT_dbgMark

void QT_dbgMark(void);

Marks the current state of memory. Leaks detected by QT_dbgReportLeaks will be relative to this reference point.

QT_dbgStopHere

void QT_dbgStopHere(void);

Empty function intended for setting a breakpoint in the debugger. Automatically called when a memory error is detected.

QT_dbgMarkChanges

QT_dbgFindChanges

void QT_dbgFindChanges(void);

Displays memory changes that occurred since the last call to QT_dbgMarkChanges.

QT_dbgReportAllocs

long QT_dbgReportAllocs(void);

Displays the list of all current allocations.

Return:

Parameters:

  • Size — Reference allocation size

Return:

QT_dbgIdStop

Sets a breakpoint on an allocation identifier. When the allocation with this identifier is made, QT_dbgStopHere will be called.

Parameters:

Return:

4. Error handling

EERRORGRAVITY

Enumeration defining the severity of an error.

Constant Description
C_WARNING Warning — the program continues normally
C_SEVERE Severe error — the program can continue but behavior is potentially degraded
C_FATAL Fatal error — the program terminates immediately

QT_Error

void QT_Error(E_ERROR_GRAVITY type, char *fcn_name, char *msg, ...);

Reports an error. The message is formatted in printf style. In case of a fatal error (C_FATAL), the program is terminated after displaying the message.

Parameters:

  • type — Error severity
  • fcn_name — Name of the function in which the error occurred
  • msg — Message in printf format
  • ... — Variable arguments for the message

Example:

QT_Error(C_FATAL, "MaFonction", "Fichier '%s' introuvable", nomFichier);

5. Strings

QT_StringUpper

char *QT_StringUpper(char *String);

Converts a string to uppercase, in place.

Parameters:

  • String — String to convert (modified directly)

Return:

  • Pointer to the modified string

QT_StringLower

char *QT_StringLower(char *String);

Converts a string to lowercase, in place.

Parameters:

  • String — String to convert (modified directly)

Return:

  • Pointer to the modified string

QT_StringClone

char *QT_StringClone(char *String);

Duplicates a string. Allocates the necessary memory and copies the content. The caller is responsible for freeing the memory with QT_Free.

Parameters:

  • String — String to duplicate

Return:

  • Pointer to the new copy, or NULL on failure

QT_StringExprCompare

boolean QT_StringExprCompare(char *String, char *Filter, boolean IgnoreCase);

Compares a string against a pattern containing wildcards. The * character matches zero or more arbitrary characters; the ? character matches exactly one character.

Parameters:

  • String — String to test
  • Filter — Comparison pattern (supports * and ?)
  • IgnoreCase — CYES to ignore case, CNO for strict comparison

Return:

  • C_YES if the string matches the pattern
  • C_NO otherwise

Example:

QT_StringExprCompare("rapport_2026.pdf", "*.pdf", C_YES);   /* C_YES */
QT_StringExprCompare("image.png", "*.jp?g", C_NO);          /* C_NO  */

QT_StringSplit

int QT_StringSplit(char *List, char **ArgValue[], char Separator);

Splits a string into an array of substrings based on a separator character. The original string is modified (separators are replaced by '\0'). The pointer array is allocated internally.

Parameters:

  • List — String to split (modified in place)
  • ArgValue — Pointer to the result array (allocated by the function)
  • Separator — Separator character

Return:

  • Number of extracted substrings

Example:

char liste[] = "un,deux,trois";
char **valeurs;
int n = QT_StringSplit(liste, &valeurs, ',');
/* n == 3, valeurs[0]=="un", valeurs[1]=="deux", valeurs[2]=="trois" */

QT_StringAppend

char *QT_StringAppend(char *Buffer, const char *String, ...);

Appends a formatted string (printf style) to the end of a dynamic buffer. The buffer is automatically reallocated if necessary. If Buffer is NULL, a new buffer is created.

Parameters:

  • Buffer — Destination buffer (can be NULL)
  • String — Format string (printf style)
  • ... — Variable arguments

Return:

  • Pointer to the buffer (possibly reallocated)

Example:

char *buf = NULL;
buf = QT_StringAppend(buf, "Bonjour %s, ", nom);
buf = QT_StringAppend(buf, "vous avez %d messages.", count);
/* buf == "Bonjour Jean, vous avez 5 messages." */
QT_Free(buf);

QT_StringInsert

char *QT_StringInsert(char *Buffer, unsigned long Position, const char *String, ...);

Inserts a formatted string (printf style) into a buffer at the specified position. The buffer is automatically reallocated if necessary.

Parameters:

  • Buffer — Destination buffer
  • Position — Insertion position (index in the string)
  • String — Format string (printf style)
  • ... — Variable arguments

Return:

  • Pointer to the buffer (possibly reallocated)

QT_StringCat

char *QT_StringCat(char *Buffer, const char *String);

Concatenates a string to the end of a dynamic buffer. Similar to QT_StringAppend but without formatting. The buffer is automatically reallocated if necessary.

Parameters:

  • Buffer — Destination buffer (can be NULL)
  • String — String to concatenate

Return:

  • Pointer to the buffer (possibly reallocated)

6. Dates and times

E_INTERVAL

Enumeration of time intervals used by the date functions.

Constant Description
C_YEAR Year
C_QUARTER Quarter
C_MONTH Month
CDAYOF_YEAR Day of the year (1-366)
C_DAY Day
C_WEEK Week
C_WEEKDAY Day of the week (0=Sunday, 6=Saturday)
C_HOUR Hour
C_MINUTE Minute
C_SECOND Second

QT_DateGMTOffset

int QT_DateGMTOffset(void);

Returns the time offset between local time and Coordinated Universal Time (GMT/UTC), in seconds.

Return:

  • Offset in seconds (positive east of Greenwich, negative west)

Example:

int offset = QT_DateGMTOffset();
/* In France (CET): offset == 3600 (1 hour) */

QT_DateCreate

time_t QT_DateCreate(int Day, int Month, int Year, int Hour, int Min, int Sec, boolean GMT);

Creates a time_t value from date and time components.

Parameters:

  • Day — Day of the month (1-31)
  • Month — Month (1-12)
  • Year — Full year (e.g.: 2026)
  • Hour — Hour (0-23)
  • Min — Minute (0-59)
  • Sec — Second (0-59)
  • GMT — CYES to interpret the values as GMT, CNO for local time

Return:

  • Corresponding time_t value

Example:

time_t noel = QT_DateCreate(25, 12, 2026, 0, 0, 0, C_NO);

QT_DatePrint

char *QT_DatePrint(char *Format, struct tm *tp, time_t DateGMT);

Formats a date/time as a string according to a strftime-style format.

Parameters:

  • Format — Format string (strftime codes: %Y, %m, %d, %H, %M, %S, etc.)
  • tp — Pointer to a tm structure (can be NULL if DateGMT is used)
  • DateGMT — time_t value (used if tp is NULL)

Return:

  • Formatted string (internal static buffer)

Common format codes:

Code Description Example
%Y 4-digit year 2026
%m Month (01-12) 03
%d Day (01-31) 08
%H Hour (00-23) 14
%M Minute (00-59) 30
%S Second (00-59) 00

Example:

char *s = QT_DatePrint("%d/%m/%Y %H:%M:%S", NULL, time(NULL));
/* s == "08/03/2026 14:30:00" */

QT_DateScan

char *QT_DateScan(char *Format, char *Date, struct tm *tp);

Parses a string representing a date according to a given format and fills a tm structure.

Parameters:

  • Format — Format string (strftime style)
  • Date — Date string to parse
  • tp — Pointer to the tm structure to fill

Return:

  • Pointer to the unparsed portion of the string
  • NULL on parsing error

Example:

struct tm tp;
QT_DateScan("%d/%m/%Y", "25/12/2026", &tp);
/* tp.tm_mday==25, tp.tm_mon==11 (0-based), tp.tm_year==126 */

QT_DateAdd

time_t QT_DateAdd(E_INTERVAL Interval, long Number, time_t DateGMT);

Adds a number of intervals to a date. The number can be negative to subtract.

Parameters:

  • Interval — Interval type (CYEAR, CMONTH, CDAY, CHOUR, CMINUTE, CSECOND, etc.)
  • Number — Number of intervals to add (negative to subtract)
  • DateGMT — Starting date

Return:

  • New date

Example:

time_t demain = QT_DateAdd(C_DAY, 1, time(NULL));
time_t il_y_a_3_mois = QT_DateAdd(C_MONTH, -3, time(NULL));

QT_DateDiff

long QT_DateDiff(E_INTERVAL Interval, time_t DateGMT1, time_t DateGMT2);

Calculates the difference between two dates, expressed in the specified unit.

Parameters:

  • Interval — Result unit (CDAY, CHOUR, CMINUTE, CSECOND, etc.)
  • DateGMT1 — First date
  • DateGMT2 — Second date

Return:

  • Difference expressed in the requested unit (DateGMT2 - DateGMT1)

Example:

long jours = QT_DateDiff(C_DAY, dateDebut, dateFin);

QT_DateTruncate

time_t QT_DateTruncate(time_t DateGMT, E_INTERVAL Interval);

Truncates a date to the lower bound of the specified interval. For example, truncating to the day sets the hour, minutes and seconds to zero.

Parameters:

  • DateGMT — Date to truncate
  • Interval — Truncation interval (CYEAR, CMONTH, CDAY, CHOUR, C_MINUTE)

Return:

  • Truncated date

Example:

time_t debut_jour = QT_DateTruncate(time(NULL), C_DAY);
time_t debut_mois = QT_DateTruncate(time(NULL), C_MONTH);

QT_DatePart

long QT_DatePart(E_INTERVAL Interval, time_t DateGMT);

Extracts a component from a date.

Parameters:

  • Interval — Component to extract (CYEAR, CMONTH, CDAY, CHOUR, CMINUTE, CSECOND, CWEEKDAY, CDAYOFYEAR)
  • DateGMT — Source date

Return:

  • Value of the requested component

Example:

long annee = QT_DatePart(C_YEAR, time(NULL));     /* 2026 */
long mois  = QT_DatePart(C_MONTH, time(NULL));    /* 1-12 */
long jour  = QT_DatePart(C_WEEKDAY, time(NULL));  /* 0=Sun, 6=Sat */

11. Files

QT_ForEachFile

boolean QT_ForEachFile(char *Path, char *Filter, boolean Recursive,
                       T_FOR_EACH_FILE_FUNC CallBack, void *Argument);

Iterates over files in a directory matching a filter and calls a callback function for each one.

Parameters:

  • Path — Path of the directory to iterate
  • Filter — Filter pattern (supports wildcards * and ?)
  • Recursive — C_YES to recursively iterate subdirectories
  • CallBack — Callback function called for each file
  • Argument — User argument passed to the callback function

Return:

  • C_YES on success
  • C_NO on failure

Callback function signature:

typedef int (*T_FOR_EACH_FILE_FUNC)(char *FileName, void *Argument);

The callback function returns 0 to continue the iteration, or a non-zero value to stop it.

Example:

int afficher_fichier(char *nom, void *arg) {
    printf("Trouve : %s\n", nom);
    return 0; /* continuer */
}

QT_ForEachFile("/var/log", "*.log", C_YES, afficher_fichier, NULL);

Complete example

The following example illustrates the use of the main UTIL module functions: initialization, configuration reading, INI file manipulation, string operations and date management.

#include "QT_std.h"
#include "QT_util.h"

int main(int argc, char *argv[])
{
    /* ---------------------------------------------- */
    /* 1. QuickTOOLS initialization                   */
    /* ---------------------------------------------- */
    QT_Init(argc, argv);

    /* Display version */
    long maj, min, rev;
    _QuickVersionNumber(&maj, &min, &rev);
    printf("QuickTOOLS %ld.%ld.%ld\n", maj, min, rev);

    /* ---------------------------------------------- */
    /* 2. Reading the configuration file              */
    /* ---------------------------------------------- */
    if (QT_ReadConfigFile("monappli") == C_YES) {
        char *serveur = QT_GetConfigKey("BaseDeDonnees", "Serveur");
        long  port    = QT_GetPrivateProfileInt("BaseDeDonnees", "Port", 3306);
        char *login   = QT_GetPrivateProfileString("BaseDeDonnees", "Login", "root");

        printf("Connexion a %s:%ld en tant que %s\n", serveur, port, login);
        QT_CloseConfig();
    }

    /* ---------------------------------------------- */
    /* 3. INI file manipulation                       */
    /* ---------------------------------------------- */
    T_INIFILE *ini = QT_IniParseFile("parametres.ini");
    if (ini == NULL) {
        ini = QT_IniCreate();
    }

    /* Read values with defaults */
    char  *theme    = QT_IniGetString(ini, "Interface", "Theme", "clair");
    long   largeur  = QT_IniGetInteger(ini, "Fenetre", "Largeur", 1024);
    double zoom     = QT_IniGetDouble(ini, "Fenetre", "Zoom", 1.0);

    printf("Theme : %s, Largeur : %ld, Zoom : %.1f\n", theme, largeur, zoom);

    /* Modify and save */
    QT_IniSetString(ini, "Interface", "Theme", "sombre");
    QT_IniSetInteger(ini, "Fenetre", "Largeur", 1920);
    QT_IniSetDouble(ini, "Fenetre", "Zoom", 1.5);
    QT_IniWriteFile(ini, "parametres.ini");
    QT_IniDestroy(ini);

    /* ---------------------------------------------- */
    /* 4. String operations                           */
    /* ---------------------------------------------- */
    /* Clone and convert to uppercase */
    char *nom = QT_StringClone("Jean Dupont");
    QT_StringUpper(nom);
    printf("Majuscules : %s\n", nom); /* JEAN DUPONT */

    /* Dynamic string building */
    char *requete = NULL;
    requete = QT_StringAppend(requete, "SELECT * FROM %s", "clients");
    requete = QT_StringAppend(requete, " WHERE ville = '%s'", "Paris");
    requete = QT_StringAppend(requete, " ORDER BY %s", "nom");
    printf("Requete : %s\n", requete);

    /* Pattern matching */
    if (QT_StringExprCompare("rapport_2026.pdf", "rapport_*.pdf", C_NO)) {
        printf("Le fichier correspond au motif\n");
    }

    /* String splitting */
    char liste[] = "rouge;vert;bleu;jaune";
    char **couleurs;
    int nb = QT_StringSplit(liste, &couleurs, ';');
    for (int i = 0; i < nb; i++) {
        printf("  Couleur %d : %s\n", i + 1, couleurs[i]);
    }

    QT_Free(nom);
    QT_Free(requete);

    /* ---------------------------------------------- */
    /* 5. Date operations                             */
    /* ---------------------------------------------- */
    /* Current date */
    time_t maintenant = time(NULL);
    printf("Aujourd'hui : %s\n",
           QT_DatePrint("%d/%m/%Y %H:%M:%S", NULL, maintenant));

    /* Create a specific date */
    time_t noel = QT_DateCreate(25, 12, 2026, 0, 0, 0, C_NO);
    printf("Noel : %s\n", QT_DatePrint("%A %d %B %Y", NULL, noel));

    /* Calculate the difference in days */
    long jours_avant_noel = QT_DateDiff(C_DAY, maintenant, noel);
    printf("Jours avant Noel : %ld\n", jours_avant_noel);

    /* Add 90 days */
    time_t dans_90_jours = QT_DateAdd(C_DAY, 90, maintenant);
    printf("Dans 90 jours : %s\n",
           QT_DatePrint("%d/%m/%Y", NULL, dans_90_jours));

    /* Truncate to the beginning of the month */
    time_t debut_mois = QT_DateTruncate(maintenant, C_MONTH);
    printf("Debut du mois : %s\n",
           QT_DatePrint("%d/%m/%Y", NULL, debut_mois));

    /* Extract components */
    printf("Annee : %ld, Mois : %ld, Jour : %ld\n",
           QT_DatePart(C_YEAR, maintenant),
           QT_DatePart(C_MONTH, maintenant),
           QT_DatePart(C_DAY, maintenant));

    /* ---------------------------------------------- */
    /* 6. Termination                                 */
    /* ---------------------------------------------- */
    QT_Terminate(0);
    return 0;
}
