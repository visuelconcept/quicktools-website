Quick Start Guide

This guide helps you get QuickSERVER up and running and create your first client in a few minutes.

Prerequisites

  • Windows 10/11 (x86 or x64)
  • QuickSERVER 2.80 installed (via the installer or manual compilation)
  • Environment variable QT_HOME pointing to the installation directory
  • Visual Studio 2015+ (to compile your client applications)

1. Verify the Installation

Open a terminal and verify that QT_HOME is set:

echo %QT_HOME%

The directory must contain:

%QT_HOME%\
├── bin\          QuickServer.exe, QuickTIMER.exe, QuickActivator.exe, QuickSDB.exe
├── lib\          qt_util.lib, qt_server.lib, qt_sdb.lib, qt_dbengine.lib
├── include\      QT_std.h, QT_util.h, QT_socket.h, QT_sdb.h, ...
├── etc\          server.dat
└── doc\          Documentation

2. Activate the License

Quick Start (without a license)

QuickSERVER automatically starts in trial mode if no license is activated. Trial mode is limited to 2 hours of continuous execution — restart the server to resume.

Activate a License

If you have a license key:

bin\QuickActivator.exe activate XXXX-XXXX-XXXX-XXXX

Activate Offline (PC without internet)

REM 1. Attempt activation (stores the key even if there is no network)
bin\QuickActivator.exe activate XXXX-XXXX-XXXX-XXXX

REM 2. Generate the request file
bin\QuickActivator.exe offline-request request.dat

REM 3. Transfer request.dat to a connected PC, upload it to the Cryptlex portal
REM 4. Download the response file, transfer it to the target PC

REM 5. Apply the response
bin\QuickActivator.exe offline-activate response.dat

Activate Trial Mode

After installation, run the trial activation script (or from the Start menu):

activate_trial.bat

Or directly:

bin\QuickActivator.exe trial

Two trial modes depending on connectivity:

Mode Condition Duration
Verified trial PC connected to the internet Duration defined by Cryptlex
Local trial PC offline (no internet) 90 days

The QuickActivator.exe trial command automatically detects connectivity:

  • If internet is available, it activates a verified trial via the Cryptlex server.
  • If the PC is offline, it automatically falls back to a 90-day local trial — no additional steps are required.

In both cases, server execution is limited to 2 continuous hours — restart the server to resume. All features (server, sdb) are enabled in trial mode.

> Note: The local trial is tied to the machine. It can only be activated once per machine (the start date is stored). If the 90-day local trial expires, only a paid license or a trial extension allows you to continue.

Extend a Local Trial (offline)

If a local trial user needs more time, you can extend their evaluation period:

bin\QuickActivator.exe trial-extend 30

This command adds 30 additional days to the current local trial. The number of days is cumulative — you can call the command multiple times. A QuickSERVER restart is required for the extension to take effect.

> Note: This command only works for local trials (offline). Verified trials (online) are managed by the Cryptlex server and cannot be extended locally. The extension does not reset the 2-hour per session limit — it only extends the calendar duration of the trial.

Check Status

bin\QuickActivator.exe status

Deactivate the License

To release the license from this PC (for example before changing machines):

bin\QuickActivator.exe deactivate

Transfer a License to Another PC

To move your license from one machine to another:

  1. On the old PC — deactivate the license to free the activation slot:
    2.    bin\QuickActivator.exe deactivate
  1. On the new PC — activate the same license key:
    2.    bin\QuickActivator.exe activate XXXX-XXXX-XXXX-XXXX

Case 1: The old PC is offline, the new PC is online

  1. On the old PC (offline) — generate a deactivation request file:
    2.    bin\QuickActivator.exe offline-deactivate deactivation_request.dat
  1. Transfer the file deactivation_request.dat to a connected PC (USB drive, internal network).
  1. On the connected PC — upload the file to the Cryptlex customer portal to finalize the deactivation.
  1. On the new PC (online) — activate the license:
    2.    bin\QuickActivator.exe activate XXXX-XXXX-XXXX-XXXX

Case 2: Both PCs are offline

  1. On the old PC (offline) — generate the deactivation request:
    2.    bin\QuickActivator.exe offline-deactivate deactivation_request.dat
  1. Transfer deactivation_request.dat to a connected PC and upload it to the Cryptlex portal.
  1. On the new PC (offline) — register the license key:
    2.    bin\QuickActivator.exe activate XXXX-XXXX-XXXX-XXXX

The online activation fails, but the key is stored.

  1. On the new PC (offline) — generate the activation request:
    2.    bin\QuickActivator.exe offline-request activation_request.dat
  1. Transfer activationrequest.dat to a connected PC, upload it to the Cryptlex portal, and download the response file activationresponse.dat.
  1. Transfer activation_response.dat to the new PC.
  1. On the new PC (offline) — apply the response:
    2.    bin\QuickActivator.exe offline-activate activation_response.dat
  1. Verify the activation:
    2.    bin\QuickActivator.exe status

> Note: Each license key has a limited number of simultaneous activations. Deactivation frees a slot, allowing the license to be activated on another machine.

3. Start QuickSERVER

bin\QuickServer.exe

Or use the launch script:

run_quickserver.bat

The server starts on port 1250 by default. The HTTP dashboard is accessible on port 1251:

http://localhost:1251

4. Diagnostics

In case of problems, use the diagnostic script:

diagnose_quickserver.bat

It checks: the QT_HOME variable, network ports (1250, 1251, 1252) and permissions.

5. First Client — Sending Messages

Create a file sender.c:

#include "QT_std.h"
#include "QT_socket.h"

#define MSG_HELLO 1001

int main(int argc, char *argv[])
{
    QT_Init(argc, argv);

    /* Connect to the local server */
    QT_SocketInit("localhost", "mon_sender", C_AUTO_RECONNECTION);

    /* Join a datagroup */
    T_DATAGROUP dg = QT_SocketJoinDG("MON_CANAL");

    /* Create a message */
    T_MESSAGE *msg = QT_SocketCreateMsg(NULL, NULL, 4096);
    msg->Header.Type = MSG_HELLO;

    /* Write typed data */
    QT_SocketWriteMsgString(msg, "Bonjour depuis le sender!");
    QT_SocketWriteMsgDouble(msg, 3.14159);
    QT_SocketWriteMsgLong(msg, 42);

    /* Send */
    QT_SocketSendMsg(msg, dg);
    printf("Message envoye!\n");

    /* Clean up */
    QT_SocketDestroyMsg(msg);
    QT_SocketTerminate();
    return 0;
}

6. First Client — Receiving Messages

Create a file receiver.c:

#include "QT_std.h"
#include "QT_socket.h"
#include <stdio.h>

#define MSG_HELLO 1001

int main(int argc, char *argv[])
{
    QT_Init(argc, argv);

    /* Connect */
    QT_SocketInit("localhost", "mon_receiver", C_AUTO_RECONNECTION);

    /* Join the same datagroup */
    T_DATAGROUP dg = QT_SocketJoinDG("MON_CANAL");

    printf("En attente de messages...\n");

    /* Reception loop */
    while (1) {
        T_MESSAGE *msg = QT_SocketWaitMsg(MSG_HELLO, 30);
        if (msg) {
            char *texte = QT_SocketReadMsgString(msg);
            double valeur;
            long entier;
            QT_SocketReadMsgDouble(msg, &valeur);
            QT_SocketReadMsgLong(msg, &entier);

            printf("Recu: %s (%.5f, %ld)\n", texte, valeur, entier);
        } else {
            printf("Timeout - aucun message recu.\n");
        }
    }

    QT_SocketTerminate();
    return 0;
}

7. Compile

With Visual Studio (command line)

REM Initialize the Visual Studio environment
call "C:\Program Files\Microsoft Visual Studio\2022\Community\VC\Auxiliary\Build\vcvarsall.bat" x86

REM Compile the sender
cl /nologo /I"%QT_HOME%\include" sender.c /link /LIBPATH:"%QT_HOME%\lib" qt_server.lib qt_util.lib ws2_32.lib advapi32.lib

REM Compile the receiver
cl /nologo /I"%QT_HOME%\include" receiver.c /link /LIBPATH:"%QT_HOME%\lib" qt_server.lib qt_util.lib ws2_32.lib advapi32.lib

With the QuickTools Build System

Place your sources in the tests/ directory and add a build script in build_scripts/.

9. First SDB Client — Shared Variables

#include "QT_std.h"
#include "QT_socket.h"
#include "QT_sdb.h"

void on_change(T_SDB inst, char *var, E_FIELD type,
               T_FIELD_VALUE *val, void *arg)
{
    printf("  >> %s = %.2f\n", var, val->Float);
}

int main(int argc, char *argv[])
{
    QT_Init(argc, argv);
    QT_SocketInit("localhost", "client_sdb", C_AUTO_RECONNECTION);

    /* Connect to SDB instance 1 */
    QT_SdbConnect("localhost", 1);
    QT_SdbReadStructure(1);

    /* Create a variable */
    QT_SdbAddVariable(1, "TEMPERATURE", C_DOUBLE, 0.0);
    QT_SdbCommit(1);

    /* Watch for changes */
    QT_SdbWatchVariable(1, "TEMPERATURE", on_change, NULL);

    /* Modify the value */
    QT_SdbSetDouble(1, "TEMPERATURE", 22.5);
    QT_SdbCommit(1);

    /* Wait for notifications */
    QT_SocketWaitMsg(0, 5);

    QT_SdbDisconnect(1);
    QT_SocketTerminate();
    return 0;
}

Compilation:

cl /nologo /I"%QT_HOME%\include" sdb_client.c /link /LIBPATH:"%QT_HOME%\lib" qt_sdb.lib qt_server.lib qt_util.lib ws2_32.lib advapi32.lib

10. First DbEngine Client — SQL Access

#include "QT_std.h"
#include "QT_util.h"
#include "Engine.h"

int main(int argc, char *argv[])
{
    QT_Init(argc, argv);

    /* Open a SQLite database */
    T_SQL *db = Sql_Init("sqlite");
    Sql_Connect(db, NULL, "test.db", NULL, NULL);

    /* Create a table */
    Sql_Execute(db, "CREATE TABLE IF NOT EXISTS capteurs "
                    "(id INTEGER PRIMARY KEY, nom TEXT, valeur REAL)");

    /* Insert data */
    Sql_Execute(db, "INSERT INTO capteurs (nom, valeur) VALUES ('%s', %f)",
                Sql_String(db, "Capteur Nord"), 42.5);

    /* Read data */
    T_SQL_RESULT *r = Sql_Execute(db, "SELECT * FROM capteurs");
    while (Sql_Fetch(r)) {
        T_SQL_FIELD *nom = Sql_GetField(r, "nom", 0);
        T_SQL_FIELD *val = Sql_GetField(r, "valeur", 0);
        printf("%s = %.1f\n", nom->Value.String, val->Value.Double);
    }
    Sql_FreeResult(r);

    Sql_Disconnect(db);
    Sql_Close(db);
    return 0;
}

Compilation:

cl /nologo /I"%QT_HOME%\include" /DUSE_SQLITE sql_client.c /link /LIBPATH:"%QT_HOME%\lib" qt_dbengine.lib qt_util.lib
