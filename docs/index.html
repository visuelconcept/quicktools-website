QuickTOOLS 2.80 — Documentation technique

Présentation

QuickTOOLS est un framework middleware développé par Visuel Concept SAS depuis 1994. Il fournit une infrastructure de communication client-serveur temps réel, une base de données partagée distribuée et un accès SQL multi-bases — le tout en C natif pour Windows.

├──────────┬──────────┬───────────────────────┤
│ SERVER   │   SDB    │       DbEngine        │

Concepts clés

Datagroups (SERVER)

Un datagroup est un canal de communication nommé. Les clients s'abonnent (join) à un datagroup pour recevoir les messages qui y sont publiés. C'est un modèle publish/subscribe :

T_DATAGROUP dg = QT_SocketJoinDG("PRODUCTION");
QT_SocketSendMsg(msg, dg);  /* tous les abonnés reçoivent le message */

Messages typés (SERVER)

Chaque message possède un type (entier ≥ 1000 pour les messages utilisateur) et peut contenir des données binaires sérialisées ou du texte ASCII. Le format binaire permet d'écrire/lire des champs typés (char, short, long, float, double, string) de manière portable entre architectures.

Variables partagées (SDB)

QuickSDB gère un espace de variables typées synchronisées en temps réel :

QT_SdbAddVariable(instance, "TEMPERATURE", C_DOUBLE, 0.0);
QT_SdbSetDouble(instance, "TEMPERATURE", 42.5);
QT_SdbWatchVariable(instance, "TEMPERATURE", callback, arg);

Les modifications sont automatiquement propagées à tous les clients abonnés.

Abstraction SQL (DbEngine)

Une API unique pour accéder à 6 bases de données différentes :

T_SQL *db = Sql_Init("sqlite");
Sql_Connect(db, NULL, "mabase.db", NULL, NULL);
T_SQL_RESULT *r = Sql_Execute(db, "SELECT * FROM clients WHERE nom = '%s'",
                               Sql_String(db, nom));
while (Sql_Fetch(r)) {
    T_SQL_FIELD *f = Sql_GetField(r, "nom", 0);
    printf("%s\n", f->Value.String);
}
Sql_FreeResult(r);
Sql_Disconnect(db);
Sql_Close(db);

Formats des livrables

Architectures

Les bibliothèques et exécutables sont disponibles en deux architectures :

Architecture Cible Flag compilateur
x86 Windows 32 bits (défaut)
x64 Windows 64 bits /D_WIN64 /DWIN64

Types de bibliothèques

Chaque bibliothèque (qt_util, qt_server, qt_sdb, qt_dbengine) est disponible en deux formats :

Format Extension Description Usage
Statique .lib Code intégré dans l'exécutable à l'édition de liens Applications autonomes, pas de dépendance DLL
Partagée (DLL) .dll + .lib (import) Code chargé dynamiquement à l'exécution Partage mémoire entre modules, mise à jour sans recompilation

Runtime C (liaison multithread)

Le choix du runtime C détermine comment votre application se lie à la bibliothèque standard Microsoft :

Option Flag Runtime Bibliothèque liée
Multithread DLL (défaut) /MD Runtime partagé (msvcrt.dll) msvcrt.lib
Multithread statique /MT Runtime embarqué dans l'exécutable libcmt.lib

Les deux modes sont multithread (/D_MT). La différence est :

  • /MD : plus petit exécutable, nécessite msvcrt.dll (présent sur tout Windows)
  • /MT : exécutable autonome, pas de dépendance externe au runtime C

Important : Tous les modules liés dans un même exécutable doivent utiliser le même runtime (/MD ou /MT). Ne mélangez pas les deux.

Compilation individuelle

Pour compiler une seule combinaison :

build_all.bat release x86 static       # x86, statique, /MD (défaut)
build_all.bat release x64 shared       # x64, DLL, /MD
set QT_MULTITHREAD=0 && build_all.bat  # runtime /MT

Gestion des licences

QuickTOOLS 2.80 utilise Cryptlex pour la gestion des licences. L'outil en ligne de commande QuickActivator.exe permet d'activer, désactiver et vérifier les licences.

Prérequis réseau et communication

Le système de licence Cryptlex établit des connexions HTTPS sortantes vers les serveurs Cryptlex (api.cryptlex.com) pour :

  • Activation / désactivation de licence
  • Vérification périodique de validité (à chaque démarrage du serveur et toutes les 30 minutes pendant l'exécution)
  • Synchronisation des dates d'expiration et renouvellements de souscription
  • Activation du trial vérifié

Ports et protocoles :

Protocole Port Direction Usage
HTTPS 443 Sortant Communication avec api.cryptlex.com

Si le PC n'a pas accès à internet (réseau isolé, firewall), le système fonctionne en mode dégradé : - Trial local (90 jours) au lieu du trial vérifié - Activation hors ligne via échange de fichiers (voir scénario 5) - Période de grâce pour les licences déjà activées (la licence reste valide un certain temps sans contact avec le serveur Cryptlex)

Firewall : Si un proxy ou un pare-feu filtre les connexions sortantes, autorisez l'accès HTTPS (port 443) vers api.cryptlex.com.

Stockage des licences

Les données de licence sont stockées localement sur la machine par le SDK Cryptlex :

Donnée Emplacement Description
Données d'activation Registre Windows (HKCU) Clé de licence, empreinte machine, jetons d'activation
Données de trial local Registre Windows (HKCU) Date de début du trial, durée restante
Product.dat Intégré dans l'exécutable Clé publique du produit (vérification cryptographique)

Les données de licence sont liées à la machine (empreinte matérielle). Elles ne sont pas transférables par simple copie de fichiers — utilisez la procédure de transfert (désactivation + réactivation).

Note : Aucun fichier license.dat n'est nécessaire. Le système Cryptlex remplace l'ancien système de licences par fichier utilisé dans les versions précédentes de QuickTOOLS.

Types de licences

Type Description
Trial Licence d'évaluation limitée dans le temps, avec arrêt automatique du serveur après 2 heures d'exécution continue. Si aucune licence n'est trouvée au démarrage, QuickSERVER démarre automatiquement en mode trial.
Subscription Licence à durée déterminée, renouvelable par souscription
Perpetual Licence permanente sans date d'expiration

Features

Les licences sont associées à des features qui déterminent les produits accessibles :

Feature Produit
server QuickSERVER — serveur de messages
sdb QuickSDB — base de données partagée

Activation en ligne

QuickActivator.exe activate XXXX-XXXX-XXXX-XXXX

Activation hors ligne (PC non connecté à internet)

  1. Définir la clé de licence (échouera car pas d'internet) : batch QuickActivator.exe activate XXXX-XXXX-XXXX-XXXX
  2. Générer une requête d'activation : batch QuickActivator.exe offline-request activation_request.dat
  3. Transférer le fichier activation_request.dat vers un PC connecté et l'uploader sur le portail client Cryptlex
  4. Télécharger le fichier de réponse et le transférer sur le PC cible
  5. Appliquer la réponse : batch QuickActivator.exe offline-activate activation_response.dat

Désactivation

QuickActivator.exe deactivate

Pour une désactivation hors ligne :

QuickActivator.exe offline-deactivate deactivation_request.dat

Vérifier le statut

QuickActivator.exe status

Démarrage automatique en mode trial

Si aucune licence valide n'est trouvée au démarrage, QuickSERVER tente automatiquement : 1. Une activation trial vérifiée (nécessite internet) 2. Un trial local (sans internet, 90 jours)

En mode trial, l'exécution du serveur est limitée à 2 heures continues. Redémarrez le serveur pour reprendre.

Vérification cyclique de la licence

QuickSERVER vérifie la validité de la licence toutes les 30 minutes pendant son exécution. Cette vérification s'applique à tous les modes (trial, souscription, perpétuel) et permet :

  • Activation à chaud : activer une licence payante avec QuickActivator.exe pendant que le serveur tourne en mode trial — la limite de 2 heures est supprimée automatiquement sans redémarrage.
  • Renouvellement à chaud : lorsqu'une souscription est renouvelée, le serveur met à jour le statut automatiquement.
  • Expiration détectée : si la licence expire ou est révoquée pendant l'exécution, le serveur s'arrête proprement.

Règles de sécurité pour le mode trial :

Les opérations suivantes nécessitent un redémarrage obligatoire du serveur pour prendre effet :

Opération Effet sans redémarrage Après redémarrage
Trial expiré (2h atteintes) Serveur arrêté Nouveau cycle de 2h
trial-extend (prolongation) Aucun effet sur la session en cours Nouvelle durée calendaire prise en compte
Licence dégradée en trial (expiration souscription) Serveur arrêté Redémarre en mode trial (2h)

Cela empêche de prolonger indéfiniment une session trial sans redémarrer le serveur.

Résumé : seule la transition trial → licence payante est prise en compte à chaud (sans redémarrage). Toutes les autres transitions impliquant un trial nécessitent un redémarrage.

Les changements de statut sont tracés dans les logs du serveur.

Scénarios d'utilisation

Scénario 1 : Premier lancement (évaluation)

Vous venez d'installer QuickSERVER et n'avez pas encore de licence.

  1. Lancez QuickSERVER normalement : batch bin\QuickServer.exe
  2. Le serveur détecte l'absence de licence et active automatiquement le mode trial.
  3. Si le PC est connecté à internet : trial vérifié via Cryptlex
  4. Si le PC est hors ligne : trial local (90 jours maximum)
  5. Le serveur fonctionne normalement avec toutes les features activées (server + sdb).
  6. Limitation : le serveur s'arrête automatiquement après 2 heures d'exécution continue. Redémarrez-le pour reprendre.

Astuce : Le mode trial permet de tester l'intégralité des fonctionnalités avant d'acheter une licence.

Scénario 2 : Passer du trial à une licence perpétuelle

Vous utilisez le mode trial et souhaitez acheter une licence permanente.

  1. Obtenez une clé de licence perpétuelle auprès de Visuel Concept.
  2. Activez la licence (le trial est automatiquement remplacé) : batch bin\QuickActivator.exe activate XXXX-XXXX-XXXX-XXXX
  3. Vérifiez le statut : batch bin\QuickActivator.exe status La sortie indique Type: perpetual et Expiration: perpetuelle.
  4. Le serveur détecte automatiquement la nouvelle licence sous 30 minutes et supprime la limite de 2 heures. Aucun redémarrage nécessaire.

Note : Il n'est pas nécessaire de désactiver le trial avant d'activer une licence. L'activation remplace automatiquement le mode précédent.

Scénario 3 : Passer du trial à une licence par souscription

  1. Obtenez une clé de licence par souscription auprès de Visuel Concept.
  2. Activez la licence : batch bin\QuickActivator.exe activate XXXX-XXXX-XXXX-XXXX
  3. Vérifiez le statut : batch bin\QuickActivator.exe status La sortie indique Type: subscription avec la date d'expiration et le nombre de jours restants.
  4. Le serveur détecte automatiquement la nouvelle licence sous 30 minutes et supprime la limite de 2 heures. Aucun redémarrage nécessaire.

Scénario 4 : Renouvellement d'une souscription

Votre souscription arrive à expiration.

  1. Renouvelez votre souscription auprès de Visuel Concept (la clé de licence reste la même).
  2. La vérification cyclique (toutes les 30 minutes) met automatiquement à jour la date d'expiration depuis Cryptlex (nécessite internet). Aucun redémarrage nécessaire.
  3. Vérifiez le renouvellement : batch bin\QuickActivator.exe status La date d'expiration et le nombre de jours restants sont mis à jour.

PC hors ligne : Si le PC n'a pas accès à internet, la licence reste valide pendant une période de grâce. Connectez le PC à internet avant l'expiration de cette période pour synchroniser le renouvellement.

Scénario 5 : Activation hors ligne (PC sans internet)

Le PC cible n'est pas connecté à internet.

  1. Sur le PC cible — tentez l'activation (enregistre la clé localement) : batch bin\QuickActivator.exe activate XXXX-XXXX-XXXX-XXXX L'activation en ligne échoue, mais la clé est mémorisée.

  2. Sur le PC cible — générez le fichier de requête : batch bin\QuickActivator.exe offline-request activation_request.dat

  3. Transférez le fichier activation_request.dat vers un PC connecté (clé USB, réseau interne, etc.).

  4. Sur le PC connecté — rendez-vous sur le portail client Cryptlex :

  5. Uploadez le fichier activation_request.dat

  6. Téléchargez le fichier de réponse activation_response.dat

  7. Transférez le fichier activation_response.dat vers le PC cible.

  8. Sur le PC cible — appliquez la réponse : batch bin\QuickActivator.exe offline-activate activation_response.dat Si le serveur tourne, la licence est détectée automatiquement sous 30 minutes. Aucun redémarrage nécessaire.

  9. Vérifiez l'activation : batch bin\QuickActivator.exe status

Scénario 6 : Transférer une licence vers un autre PC

Vous changez de machine et souhaitez déplacer votre licence.

  1. Sur l'ancien PC — désactivez la licence : batch bin\QuickActivator.exe deactivate Cela libère le slot d'activation sur Cryptlex. Si QuickSERVER tourne, il détecte la désactivation sous 30 minutes et s'arrête automatiquement.

  2. Sur le nouveau PC — activez la même clé de licence : batch bin\QuickActivator.exe activate XXXX-XXXX-XXXX-XXXX

Ancien PC hors ligne ? Utilisez la désactivation hors ligne : batch bin\QuickActivator.exe offline-deactivate deactivation_request.dat Puis uploadez deactivation_request.dat sur le portail Cryptlex depuis un PC connecté.

Scénario 7 : Licence expirée pendant l'exécution

Votre licence par souscription expire pendant que le serveur tourne.

  1. La vérification cyclique détecte l'expiration et arrête le serveur proprement en logguant un message d'avertissement.
  2. Pour retrouver un fonctionnement sans restriction :
  3. Renouvelez votre souscription auprès de Visuel Concept, puis relancez QuickSERVER (la licence est automatiquement mise à jour si le PC est connecté)
  4. Ou achetez une licence perpétuelle et activez-la : batch bin\QuickActivator.exe activate NOUVELLE-CLE-XXXX-XXXX

Scénario 8 : Prolonger un trial local (hors ligne)

Un utilisateur en trial local (90 jours) a besoin de plus de temps d'évaluation.

  1. Prolongez le trial de 30 jours supplémentaires : batch bin\QuickActivator.exe trial-extend 30 La commande est cumulative — elle peut être appelée plusieurs fois.

  2. Redémarrez QuickSERVER pour que la prolongation prenne effet. La prolongation modifie la durée calendaire du trial (nombre de jours restants) mais ne réinitialise pas la session de 2 heures en cours.

Note : Cette commande ne fonctionne que pour les trials locaux (hors ligne). Les trials vérifiés (en ligne) sont gérés par le serveur Cryptlex.

Scénario 9 : Activation à chaud d'une licence payante (serveur en production)

Le serveur tourne en mode trial ou avec une licence existante. Vous souhaitez activer une licence payante sans interrompre le service.

  1. Activez la nouvelle licence pendant que le serveur tourne : batch bin\QuickActivator.exe activate XXXX-XXXX-XXXX-XXXX

  2. Sous 30 minutes, le serveur détecte le changement et affiche dans les logs : Licence: changement de statut detecte (type: perpetual) Licence activee: limite de 2 heures supprimee.

  3. Le serveur continue de fonctionner sans interruption avec la nouvelle licence.

Important : L'activation à chaud ne fonctionne que pour passer d'un mode inférieur à un mode supérieur (trial → licence payante, ou changement de licence payante). La dégradation en mode trial (expiration de souscription) provoque un arrêt du serveur et nécessite un redémarrage.

Astuce : Pour vérifier que le changement a bien été pris en compte côté serveur, consultez le dashboard HTTP : http://localhost:1251 — la section "License Information" reflète le statut courant.

Scénario 10 : Vérifier les features disponibles

Votre licence peut inclure une ou plusieurs features (server, sdb).

bin\QuickActivator.exe status

La sortie indique les features activées : - Feature server: OUI — QuickSERVER est autorisé - Feature sdb: OUI — QuickSDB est autorisé

En mode trial, toutes les features sont activées par défaut.

Résumé des commandes
