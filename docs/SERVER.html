QuickSERVER — API de communication client-serveur

Présentation

QuickSERVER est un serveur de messages temps réel basé sur le modèle publish/subscribe. Les clients se connectent au serveur, s'abonnent à des datagroups (canaux de communication) et échangent des messages typés, binaires ou ASCII, avec compression optionnelle.

Fonctionnalités

  • Communication publish/subscribe via datagroups nommés
  • Messages binaires sérialisés (multi-types) ou ASCII
  • Compression transparente (zlib)
  • Garanties de livraison : best-effort, at-least-one, all, avec acquittement
  • Load balancing : round-robin, pondéré, trié
  • Reconnexion automatique avec restauration de contexte
  • Messages timer périodiques
  • Routage inter-serveurs
  • Dashboard HTTP intégré (port 1251)
  • Interface C et C++ (namespace quickserver)

Port par défaut

#define C_PORT_QUICKSERVER 1250

Header

#include "QT_socket.h"

Liaison

qt_server.lib qt_util.lib ws2_32.lib

Types et structures

T_DATAGROUP

typedef long T_DATAGROUP;

Handle vers un datagroup. Obtenu par QT_SocketJoinDG() ou QT_SocketGetDG().

Constantes spéciales :

Constante Valeur Description
C_LOCAL_SERVER NULL Serveur local
C_SERVER 0 Datagroup du serveur lui-même
C_DATAGROUP_ALL "_all" Broadcast à tous les clients
C_DATAGROUP_TIMER "_tmr" Messages timer

T_MESSAGE_HEADER

En-tête de chaque message (24 octets minimum).

Champ Type Accès Description
Date time_t RO Date d'émission
Source T_DATAGROUP RO Datagroup source
Dest T_DATAGROUP RW Datagroup destination
ZipLen unsigned long RO Taille compressée sur le réseau
Len unsigned long RW Taille réelle des données
Type short RW Type du message (≥ 1000 pour utilisateur)
Priority short RW Priorité
Delivery E_DELIVERY RW Mode de livraison
Timeout short RW Timeout d'acquittement
Balancing E_BALANCING RW Stratégie de load balancing
Sequence unsigned long RO Numéro de séquence unique
Binary boolean RO C_YES si données binaires

Structure complète d'un message.

Exemple complet

#include "QT_std.h"
#include "QT_socket.h"

#define MSG_TEMPERATURE 1001

int main(int argc, char *argv[])
{
    QT_Init(argc, argv);

    /* Connexion avec reconnexion automatique */
    QT_SocketInit("localhost", "capteur_01", C_AUTO_RECONNECTION_CONTEXT);

    /* Rejoindre le datagroup PRODUCTION */
    T_DATAGROUP prod = QT_SocketJoinDG("PRODUCTION");

    /* Créer et envoyer un message binaire */
    T_MESSAGE *msg = QT_SocketCreateMsg(NULL, NULL, 1024);
    msg->Header.Type = MSG_TEMPERATURE;
    QT_SocketWriteMsgString(msg, "Capteur_Nord");
    QT_SocketWriteMsgDouble(msg, 42.5);
    QT_SocketWriteMsgLong(msg, time(NULL));
    QT_SocketSendMsg(msg, prod);
    QT_SocketDestroyMsg(msg);

    /* Attendre une réponse pendant 10 secondes */
    T_MESSAGE *reply = QT_SocketWaitMsg(MSG_TEMPERATURE + 1, 10);
    if (reply) {
        char *status = QT_SocketReadMsgString(reply);
        printf("Réponse: %s\n", status);
    }

    QT_SocketTerminate();
    return 0;
}
