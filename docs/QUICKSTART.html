Guide de démarrage rapide

Ce guide vous permet de mettre en route QuickSERVER et de créer votre premier client en quelques minutes.

Prérequis

  • Windows 10/11 (x86 ou x64)
  • QuickSERVER 2.80 installé (via l'installeur ou compilation manuelle)
  • Variable d'environnement QT_HOME pointant vers le répertoire d'installation
  • Visual Studio 2015+ (pour compiler vos applications clientes)

1. Vérifier l'installation

Ouvrez un terminal et vérifiez que QT_HOME est défini :

echo %QT_HOME%

Le répertoire doit contenir :

%QT_HOME%\
├── bin\          QuickServer.exe, QuickTIMER.exe, QuickActivator.exe, QuickSDB.exe
├── lib\          qt_util.lib, qt_server.lib, qt_sdb.lib, qt_dbengine.lib
├── include\      QT_std.h, QT_util.h, QT_socket.h, QT_sdb.h, ...
├── etc\          server.dat
└── doc\          Documentation

2. Activer la licence

Démarrage rapide (sans licence)

QuickSERVER démarre automatiquement en mode trial si aucune licence n'est activée. Le mode trial est limité à 2 heures d'exécution continue — redémarrez le serveur pour reprendre.

Activer une licence

Si vous disposez d'une clé de licence :

bin\QuickActivator.exe activate XXXX-XXXX-XXXX-XXXX

Activer hors ligne (PC sans internet)

REM 1. Tenter l'activation (retient la clé même si pas de réseau)
bin\QuickActivator.exe activate XXXX-XXXX-XXXX-XXXX

REM 2. Générer le fichier de requête
bin\QuickActivator.exe offline-request request.dat

REM 3. Transférer request.dat sur un PC connecté, uploader sur le portail Cryptlex
REM 4. Télécharger le fichier de réponse, le transférer sur le PC cible

REM 5. Appliquer la réponse
bin\QuickActivator.exe offline-activate response.dat

Activer le mode trial

Après l'installation, lancez le script d'activation trial (ou depuis le menu Démarrer) :

activate_trial.bat

Ou directement :

bin\QuickActivator.exe trial

Deux modes de trial selon la connectivité :

Mode Condition Durée
Trial vérifié PC connecté à internet Durée définie par Cryptlex
Trial local PC hors ligne (pas d'internet) 90 jours

La commande QuickActivator.exe trial détecte automatiquement la connectivité :

  • Si internet est disponible, elle active un trial vérifié via le serveur Cryptlex.
  • Si le PC est hors ligne, elle bascule automatiquement sur un trial local de 90 jours — aucune manipulation supplémentaire n'est nécessaire.

Dans les deux cas, l'exécution du serveur est limitée à 2 heures continues — redémarrez le serveur pour reprendre. Toutes les features (server, sdb) sont activées en mode trial.

> Note : Le trial local est lié à la machine. Il ne peut être activé qu'une seule fois par machine (la date de début est mémorisée). Si le trial local de 90 jours expire, seule une licence payante ou une extension du trial permet de continuer.

Prolonger un trial local (hors ligne)

Si un utilisateur en trial local a besoin de plus de temps, vous pouvez prolonger sa période d'évaluation :

bin\QuickActivator.exe trial-extend 30

Cette commande ajoute 30 jours supplémentaires au trial local en cours. Le nombre de jours est cumulatif — vous pouvez appeler la commande plusieurs fois. Un redémarrage de QuickSERVER est nécessaire pour que la prolongation prenne effet.

> Note : Cette commande ne fonctionne que pour les trials locaux (hors ligne). Les trials vérifiés (en ligne) sont gérés par le serveur Cryptlex et ne peuvent pas être prolongés localement. La prolongation ne réinitialise pas la limite de 2 heures par session — elle étend uniquement la durée calendaire du trial.

Vérifier le statut

bin\QuickActivator.exe status

Désactiver la licence

Pour libérer la licence de ce PC (par exemple avant un changement de machine) :

bin\QuickActivator.exe deactivate

Transférer une licence vers un autre PC

Pour déménager votre licence d'une machine à une autre :

  1. Sur l'ancien PC — désactivez la licence pour libérer le slot d'activation :
    2.    bin\QuickActivator.exe deactivate
  1. Sur le nouveau PC — activez la même clé de licence :
    2.    bin\QuickActivator.exe activate XXXX-XXXX-XXXX-XXXX

Cas 1 : L'ancien PC est hors ligne, le nouveau PC est en ligne

  1. Sur l'ancien PC (hors ligne) — générez un fichier de requête de désactivation :
    2.    bin\QuickActivator.exe offline-deactivate deactivation_request.dat
  1. Transférez le fichier deactivation_request.dat vers un PC connecté (clé USB, réseau interne).
  1. Sur le PC connecté — uploadez le fichier sur le portail client Cryptlex pour finaliser la désactivation.
  1. Sur le nouveau PC (en ligne) — activez la licence :
    2.    bin\QuickActivator.exe activate XXXX-XXXX-XXXX-XXXX

Cas 2 : Les deux PC sont hors ligne

  1. Sur l'ancien PC (hors ligne) — générez la requête de désactivation :
    2.    bin\QuickActivator.exe offline-deactivate deactivation_request.dat
  1. Transférez deactivation_request.dat vers un PC connecté et uploadez-le sur le portail Cryptlex.
  1. Sur le nouveau PC (hors ligne) — enregistrez la clé de licence :
    2.    bin\QuickActivator.exe activate XXXX-XXXX-XXXX-XXXX

L'activation en ligne échoue, mais la clé est mémorisée.

  1. Sur le nouveau PC (hors ligne) — générez la requête d'activation :
    2.    bin\QuickActivator.exe offline-request activation_request.dat
  1. Transférez activationrequest.dat vers un PC connecté, uploadez-le sur le portail Cryptlex, et téléchargez le fichier de réponse activationresponse.dat.
  1. Transférez activation_response.dat vers le nouveau PC.
  1. Sur le nouveau PC (hors ligne) — appliquez la réponse :
    2.    bin\QuickActivator.exe offline-activate activation_response.dat
  1. Vérifiez l'activation :
    2.    bin\QuickActivator.exe status

> Note : Chaque clé de licence a un nombre limité d'activations simultanées. La désactivation libère un slot, permettant d'activer la licence sur une autre machine.

3. Démarrer QuickSERVER

bin\QuickServer.exe

Ou utilisez le script de lancement :

run_quickserver.bat

Le serveur démarre sur le port 1250 par défaut. Le dashboard HTTP est accessible sur le port 1251 :

http://localhost:1251

4. Diagnostic

En cas de problème, utilisez le script de diagnostic :

diagnose_quickserver.bat

Il vérifie : la variable QT_HOME, les ports réseau (1250, 1251, 1252) et les permissions.

5. Premier client — Envoi de messages

Créez un fichier sender.c :

#include "QT_std.h"
#include "QT_socket.h"

#define MSG_HELLO 1001

int main(int argc, char *argv[])
{
    QT_Init(argc, argv);

    /* Se connecter au serveur local */
    QT_SocketInit("localhost", "mon_sender", C_AUTO_RECONNECTION);

    /* Rejoindre un datagroup */
    T_DATAGROUP dg = QT_SocketJoinDG("MON_CANAL");

    /* Créer un message */
    T_MESSAGE *msg = QT_SocketCreateMsg(NULL, NULL, 4096);
    msg->Header.Type = MSG_HELLO;

    /* Écrire des données typées */
    QT_SocketWriteMsgString(msg, "Bonjour depuis le sender!");
    QT_SocketWriteMsgDouble(msg, 3.14159);
    QT_SocketWriteMsgLong(msg, 42);

    /* Envoyer */
    QT_SocketSendMsg(msg, dg);
    printf("Message envoye!\n");

    /* Nettoyer */
    QT_SocketDestroyMsg(msg);
    QT_SocketTerminate();
    return 0;
}

6. Premier client — Réception de messages

Créez un fichier receiver.c :

#include "QT_std.h"
#include "QT_socket.h"
#include <stdio.h>

#define MSG_HELLO 1001

int main(int argc, char *argv[])
{
    QT_Init(argc, argv);

    /* Se connecter */
    QT_SocketInit("localhost", "mon_receiver", C_AUTO_RECONNECTION);

    /* Rejoindre le même datagroup */
    T_DATAGROUP dg = QT_SocketJoinDG("MON_CANAL");

    printf("En attente de messages...\n");

    /* Boucle de réception */
    while (1) {
        T_MESSAGE *msg = QT_SocketWaitMsg(MSG_HELLO, 30);
        if (msg) {
            char *texte = QT_SocketReadMsgString(msg);
            double valeur;
            long entier;
            QT_SocketReadMsgDouble(msg, &valeur);
            QT_SocketReadMsgLong(msg, &entier);

            printf("Recu: %s (%.5f, %ld)\n", texte, valeur, entier);
        } else {
            printf("Timeout - aucun message recu.\n");
        }
    }

    QT_SocketTerminate();
    return 0;
}

7. Compiler

Avec Visual Studio (ligne de commande)

REM Initialiser l'environnement Visual Studio
call "C:\Program Files\Microsoft Visual Studio\2022\Community\VC\Auxiliary\Build\vcvarsall.bat" x86

REM Compiler le sender
cl /nologo /I"%QT_HOME%\include" sender.c /link /LIBPATH:"%QT_HOME%\lib" qt_server.lib qt_util.lib ws2_32.lib advapi32.lib

REM Compiler le receiver
cl /nologo /I"%QT_HOME%\include" receiver.c /link /LIBPATH:"%QT_HOME%\lib" qt_server.lib qt_util.lib ws2_32.lib advapi32.lib

Avec le système de build QuickTools

Placez vos sources dans le répertoire tests/ et ajoutez un script de build dans build_scripts/.

8. Tester

  1. Démarrez QuickSERVER dans un premier terminal
  2. Lancez receiver.exe dans un deuxième terminal
  3. Lancez sender.exe dans un troisième terminal

Le receiver affiche :

En attente de messages...
Recu: Bonjour depuis le sender! (3.14159, 42)

9. Premier client SDB — Variables partagées

#include "QT_std.h"
#include "QT_socket.h"
#include "QT_sdb.h"

void on_change(T_SDB inst, char *var, E_FIELD type,
               T_FIELD_VALUE *val, void *arg)
{
    printf("  >> %s = %.2f\n", var, val->Float);
}

int main(int argc, char *argv[])
{
    QT_Init(argc, argv);
    QT_SocketInit("localhost", "client_sdb", C_AUTO_RECONNECTION);

    /* Se connecter à l'instance SDB 1 */
    QT_SdbConnect("localhost", 1);
    QT_SdbReadStructure(1);

    /* Créer une variable */
    QT_SdbAddVariable(1, "TEMPERATURE", C_DOUBLE, 0.0);
    QT_SdbCommit(1);

    /* Surveiller les changements */
    QT_SdbWatchVariable(1, "TEMPERATURE", on_change, NULL);

    /* Modifier la valeur */
    QT_SdbSetDouble(1, "TEMPERATURE", 22.5);
    QT_SdbCommit(1);

    /* Attendre les notifications */
    QT_SocketWaitMsg(0, 5);

    QT_SdbDisconnect(1);
    QT_SocketTerminate();
    return 0;
}

Compilation :

cl /nologo /I"%QT_HOME%\include" sdb_client.c /link /LIBPATH:"%QT_HOME%\lib" qt_sdb.lib qt_server.lib qt_util.lib ws2_32.lib advapi32.lib

10. Premier client DbEngine — Accès SQL

#include "QT_std.h"
#include "QT_util.h"
#include "Engine.h"

int main(int argc, char *argv[])
{
    QT_Init(argc, argv);

    /* Ouvrir une base SQLite */
    T_SQL *db = Sql_Init("sqlite");
    Sql_Connect(db, NULL, "test.db", NULL, NULL);

    /* Créer une table */
    Sql_Execute(db, "CREATE TABLE IF NOT EXISTS capteurs "
                    "(id INTEGER PRIMARY KEY, nom TEXT, valeur REAL)");

    /* Insérer des données */
    Sql_Execute(db, "INSERT INTO capteurs (nom, valeur) VALUES ('%s', %f)",
                Sql_String(db, "Capteur Nord"), 42.5);

    /* Lire les données */
    T_SQL_RESULT *r = Sql_Execute(db, "SELECT * FROM capteurs");
    while (Sql_Fetch(r)) {
        T_SQL_FIELD *nom = Sql_GetField(r, "nom", 0);
        T_SQL_FIELD *val = Sql_GetField(r, "valeur", 0);
        printf("%s = %.1f\n", nom->Value.String, val->Value.Double);
    }
    Sql_FreeResult(r);

    Sql_Disconnect(db);
    Sql_Close(db);
    return 0;
}

Compilation :

cl /nologo /I"%QT_HOME%\include" /DUSE_SQLITE sql_client.c /link /LIBPATH:"%QT_HOME%\lib" qt_dbengine.lib qt_util.lib
