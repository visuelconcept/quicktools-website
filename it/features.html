Funzionalità

Tre componenti complementari per comunicazione, sincronizzazione e persistenza dei dati.

QuickSERVER — Message Broker

Sistema di messaggistica publish/subscribe per la comunicazione in tempo reale tra applicazioni distribuite.

Datagroup

Instradamento dei messaggi per gruppi nominati. I client si abbonano ai gruppi di interesse per un accoppiamento debole.

Messaggi tipizzati

Serializzazione binaria automatica dei tipi nativi: char, short, long, float, double, string. Compatibile tra architetture.

Compressione

Compressione trasparente dei messaggi per ottimizzare la larghezza di banda della rete.

Bilanciamento del carico

Strategie multiple di distribuzione del carico: round-robin, ponderato, best-effort, destinatario unico o broadcast.

Timer & messaggi periodici

QuickTIMER consente la propagazione di eventi pianificati a intervalli regolari.

Dashboard HTTP

Interfaccia web integrata (porta 1251) per il monitoraggio del server in tempo reale.

Conferma dei messaggi

Conferma di ricezione con timeout configurabili e consegna garantita.

Riconnessione automatica

Modalità di riconnessione: manuale, automatica o contestuale, per la massima resilienza.

Interconnessione server

Modalità router per architetture distribuite multi-sito con propagazione di messaggi tra server.

QuickSDB — Database Condiviso

Sistema di variabili condivise in tempo reale, sincronizzate automaticamente tra tutti i client connessi.

Variabili tipizzate

Booleano, byte, short, long, float, double, data, ora, ritardo, stringa, enum, oggetto e collezione.

Sincronizzazione trasparente

Ogni modifica viene propagata istantaneamente a tutti i client abbonati.

Modello a oggetti

Definizioni di classi con ereditarietà e membri tipizzati per un'organizzazione strutturata dei dati.

Formule algebriche

Variabili calcolate automaticamente tramite espressioni if-then-else.

Persistenza

Salvataggio e caricamento dello stato completo del database su/da disco.

Transazioni

Supporto commit/rollback per modifiche atomiche e coerenti.

Watch / Unwatch

Monitoraggio selettivo delle variabili tramite pattern per un'elaborazione efficiente guidata dagli eventi.

Gruppi di variabili

Operazioni in blocco e monitoraggio di gruppi di variabili correlate.

DbEngine — Astrazione SQL

API unificata per 6 engine di database relazionali. Cambiate engine senza modificare il codice.

Esempio di codice

Invio e ricezione di un messaggio con QuickSERVER in C:

/* Connessione e sottoscrizione */
QT_SocketInit("localhost", "MIO_CLIENT", C_AUTO_RECONNECTION);
T_DATAGROUP dg = QT_SocketJoinDG("PRODUCTION");

/* Invio di un messaggio */
T_MESSAGE msg;
QT_SocketInitHeader(&msg.Header);
QT_SocketWriteMsgLong(&msg, 42);
QT_SocketWriteMsgDouble(&msg, 23.5);
QT_SocketWriteMsgString(&msg, "OK");
QT_SocketSendMsg(&msg, dg);

/* Ricezione */
T_MESSAGE *recv = QT_SocketWaitMsg(C_MESSAGE_USER, 5);
if (recv) {
    long id; double temp;
    QT_SocketReadMsgLong(recv, &id);
    QT_SocketReadMsgDouble(recv, &temp);
    QT_SocketDestroyMsg(recv);
}
