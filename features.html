Fonctionnalités

Trois composants complémentaires pour la communication, la synchronisation et la persistance de données.

QuickSERVER — Message Broker

Système de messagerie publish/subscribe pour la communication temps réel entre applications distribuées.

Datagroups

Routage des messages par groupes nommés. Les clients s'abonnent aux groupes qui les intéressent pour un couplage lâche.

Messages typés

Sérialisation binaire automatique de types natifs : char, short, long, float, double, string. Compatible entre architectures.

Compression

Compression transparente des messages pour optimiser la bande passante réseau.

Load balancing

Distribution de charge avec stratégies multiples : round-robin, pondéré, best-effort, destinataire unique ou broadcast.

Timer & messages périodiques

QuickTIMER permet la propagation d'événements planifiés à intervalle régulier.

Dashboard HTTP

Interface web intégrée (port 1251) pour le monitoring en temps réel du serveur.

Acquittement de messages

Confirmation de réception avec timeouts configurables et livraison garantie.

Reconnexion automatique

Modes de reconnexion : manuel, automatique ou contextuel, pour une résilience maximale.

Interconnexion serveurs

Mode routeur pour architectures distribuées multi-sites avec propagation de messages inter-serveurs.

QuickSDB — Base de données partagée

Système de variables partagées en temps réel, synchronisées automatiquement entre tous les clients connectés.

Variables typées

Booléen, byte, short, long, float, double, date, heure, délai, chaîne, énum, objet et collection.

Synchronisation transparente

Toute modification est propagée instantanément à tous les clients abonnés.

Modèle objet

Définition de classes avec héritage et membres typés, pour une organisation structurée des données.

Formules algébriques

Variables calculées automatiquement par des expressions if-then-else.

Persistance

Sauvegarde et chargement de l'état complet de la base sur disque.

Transactions

Support commit/rollback pour des modifications atomiques et cohérentes.

Watch / Unwatch

Surveillance sélective des variables par patterns pour un traitement événementiel efficace.

Groupes de variables

Opérations en lot et monitoring de groupes de variables liées.

DbEngine — Abstraction SQL

API unifiée pour 6 moteurs de bases de données relationnelles. Changez de moteur sans modifier votre code.

Exemple de code

Envoi et réception d'un message avec QuickSERVER en C :

/* Envoi d'un message */ T_DATAGROUP dg = QT_SocketJoinDG(sock, "PRODUCTION"); T_MSG msg; QT_SocketInitMsg(&msg); QT_SocketPutLong(&msg, "capteur_id", 42); QT_SocketPutDouble(&msg, "temperature", 23.5); QT_SocketPutString(&msg, "status", "OK"); QT_SocketSendMsg(sock, dg, &msg); /* Réception */ if (QT_SocketReadMsg(sock, &msg, 5000) == QT_OK) { long id = QT_SocketGetLong(&msg, "capteur_id"); double temp = QT_SocketGetDouble(&msg, "temperature"); }
